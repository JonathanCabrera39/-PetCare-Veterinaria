## Reglas de Código
1. Nombres de clases: BEM o Tailwind utilities, nunca inventados
2. Componentes: Pequeños, reutilizables, un solo propósito
3. Imágenes: Siempre con `alt`, formato WebP, lazy loading
4. Formularios: Validación frontend + mensajes de error accesibles
5. No usar librerías pesadas sin justificación
